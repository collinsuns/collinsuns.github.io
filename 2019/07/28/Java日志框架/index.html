Java日志框架

发表于 2019-07-28 | 更新于 2019-08-27 | 分类于 Java | 评论数：

Java日志概述

　　对于一个应用程序来说日志记录是必不可少的一部分。线上问题追踪，基于日志的业务逻辑统计分析等都离不日志。java领域存在多种日志框架，目前常用的日志框架包括Log4j 1，Log4j 2，Commons Logging，Slf4j，Logback，Jul。

Java常用日志框架类别

  • Log4j： Apache Log4j是一个基于Java的日志记录工具。它是由Ceki Gülcü首创的，现在则是Apache软件基金会的一个项目。 Log4j是几种Java日志框架之一。
  • Log4j 2： Apache Log4j 2是apache开发的一款Log4j的升级产品。
  • Commons Logging： Apache基金会所属的项目，是一套Java日志接口，之前叫Jakarta Commons Logging，后更名为Commons Logging(简称JCL)。
  • Slf4j： 类似于Commons Logging，是一套简易Java日志门面，本身并无日志的实现。（Simple Logging Facade for Java，缩写Slf4j）。
  • Logback： 一套日志组件的实现(Slf4j阵营)。
  • Jul： (Java Util Logging),自Java1.4以来的官方日志实现。

日志演化历史

  1. 1996年早期，欧洲安全电子市场项目组决定编写它自己的程序跟踪API(Tracing API)。经过不断的完善，这个API终于成为一个十分受欢迎的Java日志软件包，即Log4j。后来Log4j成为Apache基金会项目中的一员，是当时应用最广泛的日志系统，成为了目前java日志系统事实上的标准。

  2. 2002年Java1.4发布，Sun推出了自己的日志库JUL(Java Util Logging),其实现基本模仿了Log4j的实现。在JUL出来以前，Log4j就已经成为一项成熟的技术，使得Log4j在选择上占据了一定的优势。同时也造成了目前开发者的混乱，迄今为止仍饱受诟病。

  3. 各个日志系统互相没有关联，替换和统一变的非常麻烦。A项目用log4j作为日志系统，但同时引了B项目，而B项目用jul作为日志系统，那么你的应用就得使用两个日志系统，为了搞定这个坑爹的问题，Apache推出了Jakarta Commons Logging，JCL只是定义了一套日志接口(其内部也提供一个Simple Log的简单实现)，支持运行时动态加载日志组件的实现，也就是说，在你应用代码里，只需调用Commons Logging的接口，底层实现可以是Log4j，也可以是Java Util Logging，如果没有实现Log4j和JUL也可以使用JCL自带的simpleLog。

　　　　

  1. 后来(2006年)，作为元老级日志log4j的作者Ceki Gülcü不适应Apache的工作方式，离开了Apache。然后先后创建了Slf4j(日志门面接口，类似于Commons Logging)和Logback(Slf4j的实现)两个项目，并回瑞典创建了QOS公司，QOS官网上是这样描述Logback的：The Generic，Reliable Fast&Flexible Logging Framework(一个通用，可靠，快速且灵活的日志框架)。

  2. Apache眼看有被Logback反超的势头，于2012-07重写了Log4j 1.x，成立了新的项目Log4j 2, Log4j 2具有Logback的所有特性。

Java日志框架、日志系统

  • 日志框架：只是一种日志接口，不负责具体的日志输出实现，而是通过配置来灵活的选择具体的日志系统。例如 jcl、slf4j 就是日志框架，它们只提供了接口，没有提供具体的日志系统实现；
  • 日志系统：真正的日志实现，如 log4j、jul、logback 与 log4j2 都是具体的日志系统实现。
    　　日志框架与系统，可以理解为 Java 中的接口与具体实现类。理想的日志使用模式是，代码中全部使用日志框架的 API 接口，在配置文件中配置具体的日志系统实现，这样可以通过配置来灵活切换不同的日志系统，而不用修改代码。
    　　另外我们知道日志框架与日志系统是相互配合来使用，这其中会通过适配与桥接来完成。几种日志框架与系统之间的转换关系，可以用下面这张图来表示：
    　

  • 适配：是将当前其他的日志框架或者日志系统，最终转发到 slf4j 上。例如原系统使用 jcl，那么可以通过 jcl-over-slf4j 桥接到 slf4j 上，最终日志请求会从 jcl 转发到 slf4j 上，再适配到具体的日志实现上，最终得到日志输出。例如，spring默认使用的是jcl输出日志，由于你此时并没有引入Log4j的日志系统，jcl会以jul做为日志框架。而你的应用中，采用了slf4j+log4j-core，即log4j2进行日志记录。

  • 桥接：日志框架要想实现与任意日志系统结合可能需要对应的桥接器，就是选择 slf4j 具体的日志实现类，例如使用 logback-classic 会选择 logback 作为日志实现，而使用 slf4j-log4j12 会选择 log4j 作为日志实现。

常用日志组合配置

1.slf4j + logback

logback-classic-1.0.14.jar 会自动将 slf4j-api-1.7.21.jar 和 logback-core-1.0.14.jar 也引入到项目中。

2.slf4j + log4j

slf4j-log4j12-1.7.21.jar 会自动将 slf4j-api-1.7.21.jar 和 log4j-1.2.17.jar 也添加到你的项目中

3.slf4j + java.util.logging

org.slf4j slf4j-jdk14 1.7.21

常用日志冲突解决

如果项目中已经使用了common-logging，但是想用slf4j的日志组件，可以通过适配模式，用jcl-over-slf4j.jar 把日志信息输出重定向到 slf4j-api，slf4j-api 再去调用 slf4j 实际依赖的日志组件
